# Yana Request Draft

Яна, привет!

Я сейчас собираю проект Astor Butler: AI-менеджер для ресторана/площадки, который сможет вести первичное общение с гостем по бронированию даты и организации мероприятия, собирать вводные, уточнять детали, фиксировать договоренности и готовить структурированную информацию для менеджера.

Идея такая: не заменить живого менеджера, а снять с него рутинную часть коммуникации: первичный бриф, проверка даты, сбор пожеланий, меню, формат мероприятия, количество гостей, бюджет, технические требования, документы и статусы бронирования. Диалог хотим описать через FSM-сценарии, чтобы весь процесс был управляемым, предсказуемым и пригодным для production.

Чтобы сделать это нормально, мне нужны реальные примеры материалов и документов, на которых строится работа площадки/ресторана. Можно без чувствительных данных, все персональные данные можно скрыть.

Что было бы очень полезно получить:

1. Примеры банкетных меню, пакетов, сетов, фуршетов, доп. услуг.
2. Прайсы, условия бронирования, депозиты, предоплаты, отмены, переносы дат.
3. Примеры коммерческих предложений для клиентов.
4. Примеры договоров, приложений, счетов, актов, внутренних чек-листов.
5. Описание текущего процесса: от первого сообщения клиента до финального подтверждения мероприятия.
6. Типовые вопросы клиентов и ответы менеджеров.
7. Примеры переписок или сценариев общения, можно обезличенные.
8. Какие данные менеджер обязательно должен собрать перед бронированием.
9. Какие документы нужны для подрядчиков: декор, техника, ведущие, фото/видео, кейтеринг, артисты и т.д.
10. Примеры крупных мероприятий: какая структура подготовки, какие документы, какие этапы согласования.
11. Ограничения площадки: вместимость, залы, рассадки, оборудование, тайминги, шум, алкоголь, пробковый сбор, технические условия.
12. Что считается "готовой бронью" с точки зрения ресторана/площадки.

Отдельно хочу понять, какие материалы можно использовать как референс для прототипа, а какие нельзя трогать из-за конфиденциальности. Если нужно, можем сразу договориться про NDA или работать только с обезличенными примерами.

Цель на первом этапе: собрать понятную карту процесса и документов, чтобы на ее основе сделать MVP AI-менеджера для бронирования ивентов.
