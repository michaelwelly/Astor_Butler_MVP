# FSM Docs

FSM package for scenario text companions and implementation plans.

Source hierarchy:

1. `docs/FSM_SCENARIOS_VIEWER.html` - visual source of truth, kept at root to preserve existing viewer URLs.
2. `FSM_SCENARIOS.md` - text companion for the visual map.
3. `FSM_IMPLEMENTATION_PLAN.md` - code/test implementation plan.
4. `TABLE_BOOKING.md` - table booking domain details.

If the viewer and a Markdown file disagree, update the viewer first, then sync the Markdown.
